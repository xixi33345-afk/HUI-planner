# AI_RULES.md

AI 助手在本项目中必须遵守的规则。

## 总规则

1. 修改代码前，必须先阅读相关文件和项目结构。
2. 不允许修改与任务无关的文件。
3. 不确定需求时，必须先提问，不要自行脑补。
4. 涉及删除、重构、配置、权限、密钥等高风险操作时，必须先确认。
5. 修改完成后，必须说明改了什么、为什么改、是否需要测试。
6. 如果项目有测试命令，改完后优先运行相关测试。

## 补充说明

- 高风险操作包括但不限于：删除文件/数据、大范围重构、修改构建或部署配置、变更权限、接触密钥/凭证。
- 发现规则与任务冲突时，先停下来与用户确认，不要擅自取舍。
- 重要决策和教训请分别记录到 `AI_MEMORY.md` 和 `AI_INCIDENTS.md`。
