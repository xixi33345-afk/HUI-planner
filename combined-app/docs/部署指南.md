# 部署指南：GitHub Pages + LeanCloud 云同步

## 为什么不用 Firebase？

Firebase 属于 Google，在中国大陆无法直连。本项目需要大陆和海外都能访问，因此使用 **LeanCloud 国际版**（大陆直连可用、海外正常、有免费额度）。

## 第一步：配置 LeanCloud（约 5 分钟）

1. 注册/登录 LeanCloud 国际版控制台：https://console.leancloud.app
2. 创建应用（开发版即可，免费）
3. 进入「设置 → 应用凭证」，复制三个值：**App ID**、**App Key**、**REST API 服务器地址**
4. 打开项目里的 `js/cloud-config.js`，把三个值填进去：

```js
const CLOUD_CONFIG = {
  appId: '你的 App ID',
  appKey: '你的 App Key',
  serverURL: 'https://xxxxxxxx.api.lncldglobal.com'
};
```

这三个值是公开凭证，可以放心提交到 GitHub。

5. （建议）控制台「数据存储 → 服务设置」中关闭"创建 Class"的公开权限以外的多余权限，保持默认即可；数据安全由登录账号 + ACL 保证（每条数据仅本人可读写）。

## 第二步：上传到 GitHub 并开启 Pages

1. 在 GitHub 新建公开仓库
2. 上传本文件夹全部文件（保持目录结构）
3. 仓库 → Settings → Pages → Source 选 `main` 分支根目录 → Save
4. 一两分钟后访问：`https://你的用户名.github.io/仓库名/`

## 使用方式

- 打开网站 → 「我的」→「云端同步」→ 注册邮箱+密码 → 登录
- 登录后数据自动同步；换浏览器/手机登录同一账号即可看到全部数据
- 首次登录时，本地已有数据会自动迁移到云端
- 不登录也能用，数据只存在当前浏览器

## 注意事项

- `github.io` 在大陆访问偶尔不稳定；如需稳定访问可考虑绑定自有域名或换用 Cloudflare Pages（静态文件同样直接可用）
- 改完代码重新部署后，PWA 缓存可能延迟一次刷新才更新（sw.js 已用版本号控制）
- 密码忘了暂无法自助找回（未配置邮件服务）；可在 LeanCloud 控制台 `_User` 表手动处理
